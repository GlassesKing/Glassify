# 안경 이미지·URL 크롤링·3D/왜곡 가이드

## 1. 안경 모양 보정 (다리 끝 / 옆면)

- **안경다리 끝이 알 쪽에 보일 때**: 상품에 `frontCropRatio`를 **줄이세요** (예: `0.48` ~ `0.52`). 그리면 이미지 가로 중앙만 더 짧게 잘라서 다리 끝이 잘리도록 합니다.
- **양 옆면이 너무 잘릴 때**: `frontCropRatio`를 **늘리세요** (예: `0.68` ~ `0.75`). 프론트 영역을 더 넓게 씁니다.

한 장의 이미지로 “다리 완전 숨김”과 “옆면 최대한 노출”을 동시에 맞추기는 어렵습니다.  
**권장**: 쇼핑용 이미지는 **정면만 나온 프론트 이미지**(안경다리 없음)를 쓰면, `frontCropRatio`를 `0.9` 정도로 두고 거의 전체를 써도 됩니다.

## 2. URL에서 안경 사진 크롤링 → 객체 인식 → 3D/왜곡

> “안경 사진이 있는 URL을 주면, 거기서 크롤링해 온 뒤 객체 인식으로 앞·옆면을 구분해서 3D로 만들거나 얼굴에 맞게 왜곡해 줄 수 있어?”

### 가능한 범위

| 단계 | 가능 여부 | 비고 |
|------|------------|------|
| **URL에서 이미지 수집** | △ 조건부 | 해당 사이트가 CORS/크롤링을 허용하고, 서버/백엔드에서 요청 가능해야 함. 브라우저만으로는 다른 도메인 이미지 직접 크롤링 제한 많음. |
| **객체/영역 인식** | ○ 가능 | 학습된 모델(세그멘테이션, 객체 검출)로 “안경 영역”, “앞면/옆면” 같은 영역을 추출 가능. |
| **얼굴에 맞게 왜곡** | ○ 가능 | 지금처럼 “프론트 영역만 크롭 → 얼굴 랜드마크에 맞춰 배치”하는 방식으로 확장 가능. 인식한 “앞면” 마스크만 쓰면 됨. |
| **3D로 만들기** | △ 조건부 | **정면 + 측면** 등 여러 각도 이미지가 있으면, 구조 추정(SfM 등)이나 단순 모델링으로 3D화 가능. **한 장만** 있으면 단일 이미지 3D 추정(연구 수준) 필요. |

### 구현 방향 (아이디어)

1. **이미지 소스**
   - 사용자가 **URL 목록** 또는 **이미지 파일**을 주면, 서버/백엔드에서 해당 URL을 요청해 이미지 바이너리 수집.
   - 또는 쇼핑몰/제공처 **API**로 상품 이미지 URL만 받아오고, 그 URL을 우리 서버에서 fetch하는 방식.

2. **객체/영역 인식**
   - 한 이미지 안에서 “안경” 영역을 찾기: 객체 검출 또는 세그멘테이션 모델 사용.
   - “앞면 / 옆면” 구분:  
     - 여러 각도 사진이 있으면, 각 이미지별로 영역만 잘라서 “정면용”, “측면용”으로 분류.  
     - 한 장만 있으면, 그 안에서 “가장 정면에 가까운 프레임 영역”을 마스크로 추출.

3. **얼굴에 맞게 왜곡**
   - 인식한 **앞면(프론트) 영역**만 크롭해서, 지금의 `drawGlasses` 파이프라인에 넣기.
   - 필요하면 `frontCropRatio` 대신 “인식된 마스크 기준 크롭”으로 바꾸면, URL에서 가져온 이미지도 같은 방식으로 얼굴에 붙일 수 있음.

4. **3D로 만들기**
   - **정면 + 측면** 이미지가 있을 때: 각도별로 잘라낸 텍스처를 사용해 간단한 3D 메쉬(예: 박스/실루엣)에 붙이거나, 툴로 메쉬를 만든 뒤 glb로 내보내기.
   - **한 장만** 있을 때: 단일 이미지 3D 복원은 연구/전용 모델이 필요해, 지금 프로젝트에서는 “프론트 영역 인식 → 2D 왜곡”이 현실적.

### 정리

- **URL → 이미지 수집**: 사이트 정책·CORS·백엔드 유무에 따라 가능 여부가 갈림. “이미지 URL 리스트”를 받아서 우리가 fetch하는 방식이면 구현 가능.
- **객체 인식으로 앞·옆 구분**: 가능. 그 결과로 “앞면”만 쓰면 지금 파이프라인에 붙여서 **얼굴에 맞게 왜곡**하는 건 할 수 있음.
- **3D 제작**: 여러 각도 이미지가 있으면 3D 자동/반자동 제작 가능; 한 장만 있으면 2D 왜곡이 주 방향.

원하시면 “이미지 URL 하나 주면 → (백엔드/스크립트에서) 다운로드 → 안경 영역 인식 → 프론트만 크롭해서 현재 피팅에 사용”까지의 단계를 설계해 드릴 수 있습니다.
